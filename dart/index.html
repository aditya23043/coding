Hello World

void main(){ print("Hello World"); }

Variables

void main(){ var something = "mutable string"; something = "something else"; something = true; // ERROR: cannot do implicit typecasting }

Iteration

void main(){ for(var i = 0; i < 4; i++){ print("something: $i"); } }

Const and Final

void main(){ const something = "else"; final bruh = true; }
  • Final: Runtime Constant
  • Const: Compile Time Constant

Arithmetic

void main(){ print(5 + 6); print(5 - 6); print(5 / 6); print(5 * 6); print(5 % 6); }

Concatenation / String Interpolation

void main(){ String name = "Aditya Gautam"; print("My name is " + name); print("My name is $name"); }
  • String Interpolation is preferred over concatenation for obvious reasons
  • For accessing attributes of an object..
void main(){ print("Something: ${something.name}"); }
  • This works with just variable names as well but not needed

Comment

// something /* multi * line * comment */

Type Annotation

void main(){ // automatic type inference var something = "idk"; // manual String name = "Aditya Gautam"; int age = 20; bool student = true; double pi = 3.14; }
  • Once again note that implicit type casting is NOT allowed

Null Safety

  • By default variables are "not null"
  • But they can be changed to nullable using a "?"
void main(){ int a; print(a); // ERROR int? b; print(b); // prints null }

Functions

  • Functions do not necessarily need type manual annotations
void main(){ final return_val = something(true, 10); print(return_val); } something(name, food){ return("Hi! My name is $name and I like $food."); }
  • This does not suit the requirement of the function in which we only want string
  • Furthermore, return_val is a dynamic variable since the compiler does not know the return value of the function.
  • So it is preferred to provide type annotations
String something(String name, String food){ return("Hi! My name is $name and I like $food."); }
